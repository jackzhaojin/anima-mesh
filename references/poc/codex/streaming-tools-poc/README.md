# codex streaming and tools poc

This project explores the more advanced parts of the Codex SDK:

- live event streaming with `runStreamed()`
- tagged event logging for thread, turn, item, and usage events
- command execution visibility
- file change visibility when Codex is allowed to write

## Scripts

- `npm run buffered:tools`: buffered `run()` demo that prints the final item list after the turn completes
- `npm run buffered:tools:raw-reasoning`: same buffered demo, but asks Codex CLI for raw reasoning output if available
- `npm run stream:hello`: minimal streamed hello-world
- `npm run stream:tools`: streamed inspection task that encourages command execution
- `npm run stream:write`: streamed write task that creates `output/hello-from-codex.txt`

## Event Tags

The scripts print tags such as:

- `[thread.started]`
- `[turn.started]`
- `[item.started:command_execution]`
- `[item.completed:agent_message]`
- `[turn.completed]`
- `[summary]`

These tags are generated by the local event printer so it is easy to see what Codex is doing as the turn progresses.

## Buffered vs Streamed

- `run()` gives you the full turn only after completion, but the returned `items` still show which tools or commands were used.
- `runStreamed()` gives you those same kinds of signals as they happen.
- Raw logic is not the same thing as full private chain-of-thought. In practice, you may get `reasoning` items or you may not, depending on model and CLI behavior.
